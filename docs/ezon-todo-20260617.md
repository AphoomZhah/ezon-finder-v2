# Tarea 4 — Resultados: mostrar todos los métodos de acceso

**Contexto:** hoy `ProductCard.jsx` corta los íconos de acceso con `.slice(0, 4)`. El correo pedía "2 filas de 4" (hasta 8), pero el **máximo real en el catálogo es 6 íconos** por producto. Decisión: **quitar el límite** y dejar que el grid fluya natural en filas, sin tope artificial.

```
PROMPT — Resultados: sin límite de íconos de acceso

En src/components/ProductCard.jsx, función AccessIconRow:
- Hoy hace:
    const icons = ICON_DISPLAY_ORDER
      .filter((key) => product.accessIcons?.includes(key))
      .slice(0, 4)
      .map(...)
  ELIMINA el .slice(0, 4) para mostrar TODOS los métodos del producto.
- Cambia el contenedor de los íconos para que fluya en filas en lugar de scroll horizontal.
  Hoy el wrapper usa:
    display: 'flex', flexDirection: 'row', gap: 16, flexWrap: 'nowrap',
    overflowX: 'auto', paddingBottom: 2,
  Cámbialo a un layout que envuelva en filas:
    display: 'flex', flexWrap: 'wrap', gap: 16, rowGap: 14,
  (quita overflowX: 'auto' y flexWrap: 'nowrap'). Así, si hay más de los que caben en una
  fila, pasan a la siguiente de forma natural. No impongas un número fijo de columnas;
  deja que el ancho de cada ítem (maxWidth 64 ya está) determine cuántos caben por fila.
- Mantén el resto del estilo de cada ítem (círculo 40px, label 11px) sin cambios.

Verifica npm run build y revisa visualmente una tarjeta con 5–6 métodos: deben verse todos,
envueltos en 2 filas si no caben en una. Anota en CLAUDE.md: "ProductCard: AccessIconRow sin
límite (.slice eliminado), grid fluido en filas", 2026-06-17.
No toques matcher.js, products.js ni accessIcons.js.
```

**Checklist**
- [ ] Productos con 5–6 métodos muestran todos (sin corte en 4).
- [ ] Los íconos envuelven en filas en vez de scroll horizontal.
- [ ] `npm run build` pasa.
